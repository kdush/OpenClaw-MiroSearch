# OpenClaw-MiroSearch 路线图

本文档描述项目的阶段性目标。路线图按版本演进，不构成对发布日期的承诺。

## 阶段 A：可发布基线（目标：`v0.1.0`）

范围：

- 固化 `run_research_once` 的输入输出约定
- 为 `mode` / `search_profile` 建立最小回归用例
- 明确默认生产基线：`balanced + parallel-trusted`

验收标准：

- 基础功能可用并可复现
- 对外 API 文档与行为一致

## 阶段 B：生产化（目标：`v0.2.0`）

范围：

- 支持同服务多 Key 轮转（LLM 与搜索源）
- 支持模型级 failback（主模型失败自动切换备用模型）
- 增加运行观测：429、超时、路由命中率、失败原因分布

验收标准：

- 长时间运行稳定
- 配额耗尽或单点异常时可自动降级

## 阶段 C：质量增强（目标：`v0.3.0`）

范围：

- 增加数字事实问题的交叉校验流水线
- 增加冲突检测（时间窗、统计口径、统计对象）
- 增加高置信源分级与证据对齐输出

验收标准：

- 数字类问答错误率下降
- 输出可解释，证据链清晰

## 阶段 D：生态与分发（目标：`v1.0.0`）

范围：

- 完成 OpenClaw 技能包版本化发布
- 提供部署模板（SearXNG + API 服务 + 反向代理）
- 补齐开源协作文档与版本发布规范

验收标准：

- 新环境可按文档快速部署
- 外部智能体可按技能文档直接调用
