# 安全策略

## 支持范围

当前以 `main` 与 `dev` 分支的最新版本为主要维护对象。

## 漏洞报告方式

若发现安全问题，请不要在公开 Issue 直接披露细节。建议通过私下渠道报告，并包含以下信息：

- 影响模块与文件路径
- 复现步骤
- 影响范围评估
- 建议修复方案（可选）

在仓库尚未配置专用安全邮箱前，请先通过仓库维护者可用的私下联系渠道沟通。

## 安全基线要求

- 不在仓库提交真实密钥、令牌、账号密码
- 默认使用 `.env` 管理敏感配置，提交时使用占位符
- 新增外部依赖时，评估其许可证与安全风险
- 外部输入必须做边界与错误处理

## 密钥泄露处置

若怀疑密钥已泄露，请立即：

1. 在提供方控制台吊销旧密钥
1. 生成新密钥并更新 `.env`
1. 检查日志与调用记录，确认是否存在异常请求
