# 治理说明

## 项目治理目标

- 保持接口稳定与文档一致
- 保持检索质量、成本与稳定性的平衡
- 确保开源协作过程透明、可追踪

## 角色

- 维护者（Maintainers）：负责版本发布、PR 合并、路线图推进
- 贡献者（Contributors）：通过 Issue / PR 提交改进

## 决策流程

1. 需求或问题通过 Issue 记录
1. 方案在 Issue / PR 中讨论并评审
1. 维护者基于兼容性、风险与收益做合并决策
1. 变更进入 `CHANGELOG` 与版本发布流程

## 合并原则

- 破坏性改动需提前说明迁移路径
- 新增配置项必须同步 `.env.example` 与文档
- 代码变更必须附最小可复现验证

## 版本策略

- 采用语义化版本（SemVer）
- 版本发布流程见 [`RELEASE.md`](RELEASE.md)
